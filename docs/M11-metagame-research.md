# M11 メタゲーム研究 — 噛み砕き解説

最終更新: 2026-07-20。対象読者: このリポジトリのオーナー。コードを読まずに「M11が何をやっていて、今どこまで進んで、結果をどう読むか」を把握するための文書。

---

## 1. M11とは何か(と、何ではないか)

**やっていること:** bot同士の対戦を「ものさし」にして、強いチームを自動で発掘する。

**やっていないこと:** 「botの群れが自由に戦ってメタゲームが自然発生する」ような開放的な共進化ではない。実装は、1チームを少しずつ改造して強くしていく **変異山登り(mutation hill-climbing)**。相手は固定の一線級(トーナメント上位8チーム)。

つまり正確には「**bot対戦を評価関数にした、チーム単体の最適化**」。ただし結果として、あなたの記憶にある「メタが生まれる」に近い創発が観測された(§6)。

なぜ共進化ではなく山登りなのか: 開放的な共進化はメタが振動して収束を測りにくく、"強い"の基準が曖昧になる。固定の一線級を的にすれば「既存トップに勝てるか」という明確で再現可能な物差しになる。

---

## 2. 中心のループ(1系統の動き)

「系統(lineage)」= 1つの種チームから始まる1本の探索。ループはこう:

```
種チームを用意
  └ 40回くりかえす:
       1. 現在のベストを 6通りに変異させる(子を6体つくる)
            変異の種類: 技替え / 持ち物替え / DV調整 / 努力値調整 / 種族替え / 1匹丸ごと入替
            ※ 変異した子は必ず「合法性チェック→正規化」を通る(違法な子は出さない)
       2. 6体それぞれの強さを bot対戦で測る(=適応度)
       3. 親より強い子がいれば、それを新しいベストにする(1+λ 山登り)
       4. 記録して次の反復へ
```

各系統は独立。全部で **12系統** を走らせた(§5)。

---

## 3. 「強さ」の測り方(適応度)

1チームの適応度 = **固定の一線級8チーム(トーナメント上位、プール index 0〜7)を相手に、bot対戦で何割勝つか**。

- 対戦は seed をペアにして先後を入れ替え、engine側の偏りを打ち消す。
- 相手・自分とも bot は `skuct`(状態キー付きMCTS)。探索予算は 1手あたり 300 反復(軽量・多数回すため)。
- 1回の適応度測定 = 8チーム × 8戦 = **64戦**。
- 0.5 が「互角」、それ以上なら一線級に勝ち越し。

---

## 4. 一番大事な設計判断: 「まぐれ追い(noise-chasing)」対策

これがM11の肝。64戦程度の勝率には**ばらつき(ノイズ)**がある。素朴に「親より子の勝率が高ければ採用」を繰り返すと、**本当は強くなっていないのに、たまたま良いシードを引いた子を"改善"と誤認して採用し続ける**。山登りが実質ノイズを登ってしまう。

対策として設計に組み込んだのが **hold-out検証(取り置き検証)**:

- 探索(採用の判断)で使ったシードとは**別のシード**で、最後にもう一度測り直す。
- 判定に使うのはこの**別シードでの成績(hold-out)**だけ。探索中の見かけの勝率は信用しない。

このおかげで「探索中は上がって見えたが、実は改善していなかった」系統を正しく見抜けた(§6)。この一点があるかないかで結論が真逆になり得たので、M11で最も重要な部分。

用語対応(チェックポイントJSONのフィールド):
- `seed_fitness` … 種チームの、探索中シードでの成績
- `verify.seed_fitness` … 種チームを、別シードで測り直した成績
- `verify.best_fitness` … 進化後ベストを、別シードで測り直した成績 ← **判定に使うのはこれ**

## 6. 結果 — hold-out(別シード)での順位

| 系統 | 種の強さ(別シード) | 進化後ベスト(別シード) | 読み |
|---|---|---|---|
| **rand-3**(ランダム種) | 0.208 | **0.573** | ランダムから+0.37。大成功 |
| pool-05 | 0.516 | 0.563 | 微増 |
| pool-01 | 0.661 | 0.552 | **種より悪化**(まぐれ追い) |
| pool-03 | 0.432 | 0.536 | 増 |
| pool-04 | 0.542 | 0.531 | ほぼ横ばい |
| rand-0 | 0.135 | 0.495 | 大幅増 |
| rand-2 | 0.250 | 0.479 | 増 |
| rand-1 | 0.219 | 0.438 | 増 |
| pool-00 | 0.484 | 0.490 | 横ばい |
| pool-02 | 0.578 | 0.365 | 悪化 |

### 見えた2つのこと

**(A) ランダムから始めた系統は、自力で「実メタ」に収束した ← ここが面白い**

rand-3 の種は Croconaw / Dewgong / Espeon / Lanturn / Magnemite / Seadra という、大会では見ない寄せ集め。それが40反復で
**Cloyster + Starmie + Gengar + Snorlax + Exeggutor + Suicune**
という、NC2000の定跡そのものの構築に到達した。技構成も定石(Cloysterのまきびし+だいばくはつ、Snorlaxののろい+ねむる、Suicuneのねむる+ねごと)。

人間のメタ知識を一切与えず、bot対戦の勝ち負けだけを頼りに実メタへ辿り着いた。これは「**bot対戦がメタゲームを再発見した**」に近く、あなたが記憶していた"メタが生まれる"像に最も近い成果。

**(B) 既存の強いチーム(プール種)からは、ほとんど伸びなかった**

pool-01 は探索中の見かけ上は 0.719 まで登ったが、別シードで測り直すと 0.552 で、**種(0.661)より弱い**。＝まぐれ追いを hold-out 検証が正しく捕まえた。既にプールに入っている一線級は最適化の余地が乏しく、山登りは主にノイズを登っていた。

直感的な理由: ランダム種は「伸びしろが全部」なので登れる。完成度の高いプール種は既に頂上付近で、64戦のノイズが本物の改善差を上回ってしまう。

## 7. 認定(certification)— 本当に一線級を超えるかの最終判定

§6の勝率は 300予算・64戦の**ライブ対戦**の値で、ノイズを含む。最終判定はもっと厳密にやる:

- 発掘チームについて、一線級8チームとの**チームプレビュー均衡を厳密にオフラインで解く**(`bake_preview --candidate`。M8のテーブルと同じ機構)。
- その **対T1平均均衡値** を、既存プールの基準線と比べる。
- **プール基準線 = 最強 sample-01 の 0.5968**(全595テーブルから算出した対T1平均均衡値)。

**判定ルール(オーナー決定):** 明確な大差でこの基準を超える発掘チームがあれば取り込み候補。無ければ「現状プールで十分」＝打ち切り。

暫定の読み: rand-3 のライブ 0.573 はプール最強の相当値と拮抗しており、「一線級を明確に超える新チーム」が出る公算はやや低い。ただし**厳密均衡の認定値が最終判定者**で、それが揃うまで結論は保留。
